使用 Git 進行版控

如何管理程式碼

  • 期末專案.zip
  • 期末專案（完成版）.zip
  • 期末專案（終極完成版）.zip
  • 期末專案（最終完成版）.zip
  • 期末專案（真的不會再修改版）.zip

這個作法稱之為「複製貼上大法」

這樣的作法很有明顯的缺點

每次的新版本都要重新取名子

光看檔案名稱，我們無法快速的了解：

  • 無法新版本的成因
  • 無法得知各版本之間的差異
  • 無法追蹤修改者和修改的內容

在使用複製貼上大法時

有可能意外覆蓋掉別人的檔案

Git

Git 是目前業界最流行的版本控制系統

最初目的是為更好地管理 Linux Kernel開發而設計

Git 與其他版控工具不同

Git 是分散式版控工具

也就是說 Git 不需要一台遠端伺服器

就算沒有網路，在你的電腦上也可以操作

待有網路的時候再與其它人同步即可

如果不使用 Git

還是可以持續學習 React

但能夠靈活的使用 Git

管理自己的程式碼

不會誤刪別人程式碼

如此才能與他人合作

Commit

在 Git 中

每一次的更動稱之為 Commit

多人合作開發
